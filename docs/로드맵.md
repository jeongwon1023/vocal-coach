# 보컬 코치 4단계 로드맵

분석 엔진은 완성되었습니다. 이후는 **그릇(UI·배포·사용자)** 을 만드는 단계입니다.

---

## [1단계] 웹 MVP 출시 (현재 ~ 1개월) ✅ 진행 중

**목표:** 친구·소규모 사용자에게 링크 공유

| 작업 | 상태 | 파일 |
|------|------|------|
| Streamlit 대시보드 UI | ✅ | `app.py`, `ui/dashboard.py` |
| 마이 페이지 (분석 이력) | ✅ | `ui/my_page.py` |
| GPT 코칭 연동 | ✅ | `--gpt` / 사이드바 옵션 |
| **로컬 웹 실행** | ✅ | `run_web.bat` |
| Streamlit Cloud 배포 | ⬜ **다음** | [다음-스텝.md](다음-스텝.md) |
| Supabase 준비 | ✅ 골격 | `supabase/schema.sql`, `db_store.py` |

**실행:**
```powershell
cd "C:\Users\chahy\OneDrive\바탕 화면\vocal-coach"
.\run_web.bat
```

---

## [2단계] 사용자 경험 고도화 (1~2개월)

**목표:** 재방문율 높이기

| 작업 | 설명 |
|------|------|
| Supabase 연동 | JSON 로컬 → 클라우드 DB, 사용자별 기록 |
| 로그인/회원가입 | 이메일 또는 소셜 로그인 |
| 마이 페이지 고도화 | 점수 변화 그래프, 주간 리포트 |
| WebView 앱 | 웹을 감싸서 앱스토어 출시 (개발비 최소) |

---

## [3단계] 모바일 서비스화 (2~3개월)

**목표:** 스마트폰 녹음 + 코칭

| 작업 | 설명 |
|------|------|
| Expo 앱 본격 개발 | `mobile/` 폴더 골격 → 녹음 UI 완성 |
| API 서버 배포 | `api_server.py` → Railway/Render 등 |
| 앱 ↔ 서버 연동 | 녹음 → POST /analyze → 결과 화면 |
| 푸시 알림 | 연습 시간 리마인더 (Expo Notifications) |

**지금 준비된 것:**
- `api_server.py` — REST API
- `mobile/` — Expo 프로젝트 골격

---

## [4단계] 스케일업 (3개월 이후)

**목표:** 서비스 완성도 · 수익 모델

- 구독형 모델 (월 N회 분석 + GPT 코칭)
- CNN 등 고도화 분석 모델
- 마케팅 · 인플루언서 협업

---

## 추천 순서 (오늘부터)

1. **`run_web.bat`** 로 웹 UI 확인
2. **Streamlit Cloud** 무료 배포 → 링크 공유
3. 친구 3~5명 피드백
4. 2단계 Supabase 전환 검토
5. 모바일은 **WebView** 또는 Expo 중 선택

> 굳이 처음부터 네이티브 앱을 만들 필요 없습니다.  
> 웹이 완벽하면 WebView로 앱처럼 출시할 수 있습니다.
