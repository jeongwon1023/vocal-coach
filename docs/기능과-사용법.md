# 기능과 사용법

## 1. 4단계 커리큘럼

| Stage | 이름 | 분석 내용 |
|-------|------|-----------|
| 1 | 음정 (Pitch) | 기준 멜로디 대비 센트 편차, 지속음/비브라토 구분, DTW 정렬 |
| 2 | 박자 (Rhythm) | 보컬 Energy Envelope, 어택 간격 변동계수 |
| 3 | 호흡·음색 | Energy Envelope, 고주파 배음 하락, 호흡 불일치 |
| 4 | 종합 코칭 | [결과] [원인] [해결] 형식 + GPT (선택) |

## 2. 출력물

| 파일/폴더 | 설명 |
|-----------|------|
| `pitch_result.png` | 음정 맵 (파란=안정, 빨간=이탈, 초록=기준) |
| `records/record_*.json` | 날짜별 분석 기록 |
| `charts/growth_chart.png` | 종합·Stage별 성장 곡선 |
| `clips/*.wav` | 문제 구간 집중 연습 클립 |
| `analysis_report.json` | `--json-out` 지정 시 |

## 3. 실행 방법

### A. 배치 파일 (가장 쉬움)

프로젝트 루트 **`vocal-coach`** 폴더에서 더블클릭:

| 파일 | 기능 |
|------|------|
| `run_check.bat` | 패키지·API 키 점검 |
| `run_analysis.bat` | 기본 분석 + 그래프 |
| `run_record.bat` | 기록 + 클립 + 성장그래프 + 비교 |
| `run_gpt.bat` | 위 + GPT 코칭 |
| `run_web.bat` | 웹 UI (Streamlit) |

### B. PowerShell (CLI)

```powershell
cd "c:\Users\chahy\OneDrive\바탕 화면\vocal-coach"

# 환경 점검
.\venv\Scripts\python.exe check_setup.py

# 기본 분석
.\venv\Scripts\python.exe analysis.py sample.mp3

# 전체 기능
.\venv\Scripts\python.exe analysis.py sample.mp3 ^
  --save-record --compare --export-clips --growth-chart --gpt

# 곡 제목 + 유튜브 가이드 + DTW
.\venv\Scripts\python.exe analysis.py my.mp3 --song "곡제목" --fetch-reference --save-record
```

| **마이 페이지** | `records/` 이력 테이블 · 성장 곡선 · 기록 비교 · JSON 다운로드 |

Streamlit Cloud 배포: [docs/배포-비용-가이드.md](배포-비용-가이드.md)

## 4. CLI 옵션 전체

| 옵션 | 설명 |
|------|------|
| `audio` | 분석할 파일 (생략 시 sample.mp3) |
| `--song "제목"` | 유튜브 가이드/MR 검색 |
| `--fetch-reference` | yt-dlp 다운로드 |
| `--reference file.mid` | MIDI 기준 멜로디 |
| `--guide file.wav` | 로컬 가이드 (DTW) |
| `--gpt` | GPT 코칭 |
| `--save-record` | records/ JSON 저장 |
| `--compare` | 이전 기록과 점수 비교 |
| `--export-clips` | clips/ WAV 추출 |
| `--growth-chart` | charts/growth_chart.png |
| `--json-out path.json` | JSON 저장 경로 |
| `--save path.png` | 그래프 저장 경로 |

## 5. 지금 당장 해야 할 3가지 (자동 출력)

분석 후 터미널/리포트에 **우선순위 실행 플랜**이 출력됩니다.

1. **박자**: 메트로놈 70BPM, 2마디×손뼉 5회
2. **음색/호흡**: 성대 접촉 '아-' 10분
3. **기록**: `--save-record` → 1주 후 재분석 비교

## 6. MR(반주) 포함 녹음

- 분석 시 **MR/반주 감지** 메시지 표시
- MR 포함 시 박자·음색 점수가 보수적으로 나올 수 있음
- **권장**: 이어폰으로 MR 듣고, 마이크에는 **목소리만** 녹음

## 7. GPT 설정

1. `.env` 파일:
   ```
   OPENAI_API_KEY=sk-...
   OPENAI_MODEL=gpt-4o-mini
   ```
2. `run_gpt.bat` 또는 `--gpt`

## 8. 성장 추적 워크플로

```
1주차: run_record.bat  → records/record_xxx.json
       clips/ 에서 문제 구간만 반복 연습
1주 후: run_record.bat  → --compare 로 점수 변화 확인
       charts/growth_chart.png 로 곡선 확인
```

## 10. 자주 하는 실수

| 증상 | 해결 |
|------|------|
| `run_gpt.bat` 못 찾음 | **`vocal-coach` 폴더**에서 실행 (venv 안 X) |
| `python`만 뜨고 멈춤 | `venv\Scripts\python.exe` 사용 |
| Activate.ps1 오류 | `.bat` 사용 또는 python.exe 직접 호출 |
| PowerShell 설치 안내 | 오류 아님, 무시 가능 |
| GPT 429 | OpenAI Billing 충전 |
